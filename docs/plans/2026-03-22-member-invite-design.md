# Member Invite Entry Design

**Date:** 2026-03-22

## Goal

将原本独立占用顶栏位置的邀请中心收拢到会员中心内部，同时保留现有邀请归因、用户侧邀请记录展示、管理员侧邀请统计与管理能力。

## Approved Direction

用户确认采用以下方向：

- 顶栏移除独立“邀请中心”入口
- 会员中心内部提供二级切换，使用中性文案，不强调邀请权益
- 邀请链接进入注册页时，邀请码需要自动带入表单
- 普通注册页同样展示邀请码字段，但作为选填
- 管理员后台继续展示邀请统计、排行和最近转化数据
- 用户继续查看自己邀请用户的脱敏邮箱与公司名称

## Architecture

### Dashboard Navigation

- 顶层工作台保留 `家具图册`、`室内编辑器`、`会员中心`
- `会员中心`内部承载两个分区：
  - `会员信息`
  - `邀请链接`
- 旧的 `/?tab=invite` 继续兼容，但会规范化到新的会员中心邀请分区路径

### Invite Deep Link

- 新的邀请回跳路径统一为 `/?tab=vip&section=invite`
- 邀请链接访问后：
  - 未登录用户跳到 `/signup?invited=1&code=<邀请码>`
  - 已登录用户继续沿用现有归因逻辑
- 邮箱验证完成后仍回到会员中心里的邀请链接分区

### Sign-up Form

- 新增邀请码输入框
- 邀请注册时：
  - 自动填入邀请码
  - 保持可编辑
- 普通注册时：
  - 字段显示为选填
- 注册接口优先使用表单中的邀请码，空值时再回退到 cookie 中的邀请码

### Data Visibility

- 用户侧继续展示：
  - 邀请链接
  - 邀请码
  - 邀请统计
  - 最近邀请记录中的脱敏邮箱与脱敏公司名
- 管理员后台继续展示：
  - 活跃邀请链接
  - 已注册邀请
  - 已验证邀请
  - Top 邀请人
  - 最近转化明细

## Constraints

- 不新增关于邀请权益或被邀请权益的公开说明文案
- 不重写现有邀请数据模型或后台统计逻辑
- 最小改动完成入口归位、邀请码表单化和旧路径兼容
